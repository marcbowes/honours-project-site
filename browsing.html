Browsing

Introduction

Browsing through a digital collection is a primary function of Digital Repository Systems and entails users being able to navigate through items in the digital collection based on the various categories.

This is simple to do using database management software (DBMS), but as AJAX is not able to interface with a DBMS and the definition of the project is to minimize the software footprint, the browsing system needs to be implemented from scratch.

Design

Browsing in a Digital Repository System at the lowest level is basically creating lists of items sorted by various criteria which the users can then navigate through. Thus creating a browsing system consists of two parts:

  • Sorting the data
  • Displaying the sorted lists
As AJAX is unable to write to a filestore, the sorting had to be done in a pre-processor. This essentially sub-divides the browsing system into two separate systems:
  • Pre-processor
  • Web Site

The interaction overview of the system.

Implementation

Pre-processor

As stated before, the pre-processor was needed because JavaScript is unable to write to disk.
The Pre-processors job is to to pre-sort the data by its various metadata fields, as well as performing other tasks described below, in order to allow JavaScript to offer functionality without having to access every metadata file.

The pre-processor was written in Java, to keep cross platform compatibility.
The tasks performed by the pre-processor are:

Sorting the data

  • Reading metadata for each valid file from an unknown directory structure.
  • Sorting the data based on each of the fields in the metadata.
  • Creating a representation for each file.

Writing out the lists for the site

  • Omitting fields the user does not want to browse.
  • Creating pagination.
  • Creating quick navigation links.
  • Linking the lists to the website.

Once these tasks have been completed the website can use the lists and other files output by the pre-processor to browse the collection.

Web Site

The Web site handles all the user interaction, facilitating the browsing experience. The Browsing functionality is created dynamically through AJAX.
The AJAX reads in the lists generated by the Pre-processor and displays them on the page.
The users can then navigate through these lists and access items in the lists.

The Web site also provides online integration with the central repository.
It does this by requesting an update from the central repository and then integrating the files received with the offline files in real time.

Results

Various tests were performed, namely:

  • Compatibility Testing
  • Performance Testing
  • User Testing

From these tests, the following conclusions were made:

  • The system was found to be functional on a range of Operating Systems and Web Browsers.
  • Performance was not a hindrance to the browsing experience.
  • Even inexperienced users found the system responsive and easy to use. They also believed that the system had all the necessary functionality for a simple browsing system.

Conclusion

Based on the goal of creating a lightweight and distributable digital repository system, an experimental browsing system was designed and implemented that can be used offline and online with no extra software needed other than a JavaScript compliant Web Browser.

Testing this system showed that it succeeded in being distributable, lightweight and providing the basic functionality expected of a browsing system. However, the AJAX limitations were present, in that it required a pre-processor in order to provide the browsing functionality. The nature of AJAX also limited the functionality that could be implemented.
